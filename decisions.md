# Weather App 설계 의사결정 기록 (ADR)

## ADR-001: 날씨 상태별 배경색 도입
날짜: 2026-05-11

### 배경
기존 단색 배경이 날씨 상태를 직관적으로 전달하지 못했음.
텍스트에만 의존하여 한눈에 날씨 파악이 어려웠음.

### 결정
북유럽 감성 컬러 팔레트를 날씨 상태별 배경색으로 적용.
- 맑음: #2DBD8F (에메랄드 그린)
- 비: #3B82C4 (코발트 블루)
- 눈: #9B7FD4 (라벤더)
- 흐림: #7A99A8 (스틸 블루 그레이)
- 구름조금: #2DBD8F

### 이유
현대카드 웨더 앱 분석 결과, 배경색 자체가 날씨 상태를
가장 빠르게 전달하는 방식임을 확인.
텍스트 없이도 색상만으로 날씨 인지 가능.
채도가 높되 원색적이지 않은 북유럽 톤 선택.

### 결과
Weather Icons 폰트(CDN)와 조합하여 적용 완료.
배경 위 텍스트는 흰색(#FFFFFF) 고정으로 가독성 확보.

## ADR-002: 기온 바 그래프 시각화 원칙
날짜: 2026-05-11

### 배경
주간 예보 바 그래프에서 오늘 바만 비정상적으로 길고
나머지가 납작하여 비교가 불가능했음.

### 결정
7일 전체 기온 범위를 기준으로 상대적 높이와 위치를 계산.
- 바 위치: (최저기온 - 전체최솟값) / 전체범위 × 최대높이
- 바 높이: (최고기온 - 최저기온) / 전체범위 × 최대높이
- 최소 바 높이: 20px
- 전체 바 영역 높이: 200px 고정

### 이유
현대카드 웨더에서 발견한 핵심 원칙:
바의 위치 자체가 기온 정보를 담아야 함.
높이만이 아니라 위치로도 날씨를 비교할 수 있는 구조.

### 결과
7일 기온이 공통 수직축을 공유하여
날짜 간 기온 비교가 시각적으로 가능해짐.

## ADR-003: 텍스트 가독성 기준 수립
날짜: 2026-05-11

### 배경
라이트/다크 모드에서 연한 회색 텍스트가 배경과 구분되지 않아
주간 예보, 시간대별 섹션의 가독성이 낮았음.

### 결정
텍스트 색상 규칙을 명시적으로 수립:
- 컬러 배경 위: #FFFFFF 고정
- 일반 섹션 본문: 라이트 #1A1A1A / 다크 #F0F0F0
- 보조 텍스트: 라이트 #444444 / 다크 #CCCCCC
- 연한 회색(#999999 이상) 금지
글자 크기 기준:
- 현재 기온: 48px / 섹션 제목: 18px
- 날씨 상태: 16px / 보조 텍스트: 14px / 최소: 13px

### 이유
모드 전환에 관계없이 일관된 가독성 보장.
명시적 규칙이 없으면 수정할 때마다 다시 깨짐.

### 결과
라이트/다크 모드 모두에서 텍스트 가독성 확보.
